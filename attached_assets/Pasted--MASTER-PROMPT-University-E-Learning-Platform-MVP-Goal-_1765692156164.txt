✅ MASTER PROMPT – University E-Learning Platform (MVP)
Goal

Build a scalable university e-learning platform (MVP first) for paid academic courses (external payment), targeting university students across multiple colleges, with strict role-based access control, approval workflows, locked content, and college-based theming.

Context

The platform is an online educational system for universities, not certificates.
Students do not self-enroll or pay inside the platform.
Teachers upload courses, but publishing requires approval by Admin or Super Admin.
Students can browse all courses and open lessons, but lesson content is locked unless enrolled.
Communication between students and teachers is email-only (mailto).
The platform starts minimal and is designed to scale later.

User Roles

There are 4 roles with strict permissions:

STUDENT

TEACHER

ADMIN

SUPER_ADMIN

Colleges

The platform starts with 3 colleges:

Pharmacy

Engineering

IT

Each college:

Has its own courses

Has its own UI theme

Is selectable by students during signup

Permissions & Rules
Student

Can view shared home page

Can browse all published courses

Can view course details and lesson list

Can open lesson pages

If not enrolled:

Lesson content is locked

Show lock UI (no content leakage)

Show “Contact Teacher” button (mailto)

If enrolled:

Full access to lesson content

Cannot enroll or unenroll

Cannot message inside platform

Teacher

Can create and edit own courses

Courses start as DRAFT

Can submit course for approval → status becomes PENDING_APPROVAL

Cannot publish directly

Can create/edit lessons

Can enroll students to own courses (add only)

Cannot remove students from courses

Can see number of students per course

Admin

Can view all teachers

Can see teacher stats:

Number of courses

Number of enrolled students

Can approve or reject courses:

Approve → status = PUBLISHED

Reject → status = REJECTED (optional reason)

Can manage users (delete / disable – scaffold endpoints at minimum)

Cannot change user roles unless explicitly allowed later

Super Admin

Full system access

Can:

Create / update / delete colleges

Approve or reject courses

Assign or change user roles:

Student ↔ Teacher ↔ Admin ↔ Super Admin

Enroll students into any course

Manage all users and content
