PROMPT للـAgent: Implement “Forgot Password” + “Reset Password” for internal auth

Goal

Add a secure “Forgot password” flow to the unified login system (email/password), allowing users to reset their password via email link with a time-limited, single-use token.

Requirements
1) UI صفحات

On Login page, add a link: “Forgot password?”

Create pages:

/forgot-password

input: email

submit button

success message (always the same whether email exists or not)

/reset-password?token=...

inputs: new password + confirm password

submit button

show errors for invalid/expired token

success redirects back to login

2) API Endpoints

POST /api/auth/forgot-password

body: { email }

Always return: “If an account exists, we sent a reset link.”

If user exists, generate token, store hash, set expiry (e.g., 30 minutes), send email with reset link.

POST /api/auth/reset-password

body: { token, newPassword }

Validate token:

hash matches a stored record

not expired

not used

Update user password hash

Mark token as used (usedAt)

Invalidate any active sessions if you support sessions (optional but recommended)

3) Database / Models

Create PasswordResetToken table/model:

id

userId

tokenHash

expiresAt

usedAt (nullable)

createdAt

Store only tokenHash (never store raw token).

4) Token Rules

Token length: 32+ bytes random

Expiry: 30 minutes default

One-time use

Rate limit: basic throttling per email/IP to prevent abuse

5) Email Delivery

Implement email sending via SMTP or provider.

In development mode, it’s acceptable to log the reset link to the console if no SMTP is configured, but in production it must send email.

Make email sender configurable via env vars.

6) Password Policy

Minimum length 8 (or 10)

Confirm password must match

Hash with bcrypt/argon2

Show friendly validation messages

Acceptance Criteria

User can request password reset from login.

Reset email link works and allows setting a new password.

Expired/invalid/used token is rejected.

Email existence is not disclosed.

Works on external/published app (not dependent on Replit Auth).

Deliverables

File paths for:

forgot/reset pages

API handlers

DB migration/model

email utility

Manual test steps
